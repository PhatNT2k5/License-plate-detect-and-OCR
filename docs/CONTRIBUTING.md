# Contributing Guidelines

Thank you for considering contributing to the License Plate Recognition System! We welcome contributions from the community.

## How to Contribute

1. **Fork the Repository**
   - Click the "Fork" button at the top right of the repository page
   - Clone your fork locally: `git clone https://github.com/PhatNT2k5/license-plate-recognition.git`

2. **Create a Branch**
   - Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name`
   - Use descriptive branch names: `feature/new-model`, `bugfix/issue-description`, `docs/update-readme`

3. **Make Your Changes**
   - Follow the coding standards and style guidelines
   - Write clear, descriptive commit messages
   - Add tests for new functionality when applicable
   - Update documentation as needed

4. **Submit a Pull Request**
   - Push your branch to your fork: `git push origin feature/your-feature-name`
   - Open a pull request against the main repository's `main` branch
   - Provide a clear description of your changes and their purpose
   - Reference any related issues

## Coding Standards

- **Python**: Follow PEP 8 style guidelines
- **Comments**: Write clear, concise comments explaining why, not what
- **Naming**: Use descriptive names for variables, functions, and classes
- **Imports**: Group imports logically (standard library, third-party, local)
- **Error Handling**: Use appropriate exception handling and logging

## Pull Request Process

1. Ensure your code passes any existing tests
2. Update the README.md if your changes affect usage or installation
3. The maintainers will review your PR and may request changes
4. Once approved, your PR will be merged

## Reporting Issues

When reporting bugs or suggesting features, please include:
- A clear, descriptive title
- Steps to reproduce the issue (for bugs)
- Expected vs actual behavior
- Screenshots or logs if applicable
- Environment details (Python version, OS, etc.)

## Code of Conduct

Please be respectful and considerate of others when contributing to this project.
Harassment, discrimination, or disruptive behavior will not be tolerated.

## Getting Help

If you need help with your contribution:
- Check the existing documentation
- Look at similar contributions in the repository
- Ask questions in the issue tracker
- Contact the maintainers directly

Thank you for contributing to make this project better!
